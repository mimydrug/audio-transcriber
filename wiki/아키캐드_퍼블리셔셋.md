## Overview

Publisher Sets are the primary way to batch-issue drawings from ArchiCAD. A well-structured Publisher Set means you can issue a full drawing set in one click, with consistent naming, format, and output path.

---

## Workflow: Issuing PDFs from a Publisher Set

**Learned: 2026-04-13**

### Steps

1. Open **File → Publish** (or the Publisher palette via the menu).
2. Select the Publisher Set for the current issue (e.g. `DA Issue`, `CD Issue`, `RFI Response`).
3. Check that the **Output Path** at the top is pointing to the correct project folder — update it if the project has moved.
4. Review the sheet list — toggle off any sheets not included in this issue using the checkbox next to each layout.
5. Set the **Format** to PDF (or DWG if issuing to consultants).
6. Click **Publish** — ArchiCAD will generate all files in one pass.

### Tips

- Keep one Publisher Set per issue type (DA, CD, Tender, Construction) rather than modifying a single set each time.
- Use **Auto-Naming** with tokens like `{Layout Name} - {Revision}` to keep file names consistent without manual renaming.
- If sheets are missing from the set, check that the Layout is linked to the correct Publisher Set in the Layout Book properties.
- For DWG exports, always check the **Translation** settings — confirm which pen set and layer mapping you're using before issuing to consultants.

### Common Issues

| Problem | Fix |
|---------|-----|
| Output path missing / wrong folder | Re-link via the path field at the top of the Publisher palette |
| Wrong page size in output PDF | Check Layout properties → Page Setup |
| Sheet numbering out of order | Drag-reorder layouts in the Publisher Set tree |
